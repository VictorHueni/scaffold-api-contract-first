# Reference

Deep reference material for architects, integration specialists, and anyone planning beyond v1 of the scaffold. These are catalogs and specifications, not how-to guides — for hands-on tasks see [`../guides/`](../guides/).

## Catalogs

| Reference | When to read it |
|---|---|
| [Contract-First Maturity Model](contract-first-maturity-model.md) | Planning organizational adoption. Levels L1 (Foundation: lint, mock, generate) → L4 (Platform: org-wide registry, governance). Use it to map where you are today and what to invest in next. |
| [EIP Patterns for API Mediation](eip-patterns-for-api-mediation.md) | Designing API mediation, integration flows, or ESB-style transformations. 20 Enterprise Integration Patterns (Hohpe & Woolf) translated to API mediation contexts. Foundation for [PRD 0002](../product-specs/0002_prd_integration-mapping-as-code.md). |
| [OpenAPI Overlay Specification](openapi-overlay-specification.md) | Annotating or extending OpenAPI specs you don't own (vendor APIs, generated specs). Full reference for OpenAPI Overlay Spec v1.1.0. Foundation for [PRD 0002](../product-specs/0002_prd_integration-mapping-as-code.md). |
