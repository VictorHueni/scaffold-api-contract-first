# Documentation

Project documentation for the contract-first scaffold. The scaffold itself lives in [`../scaffold-api/`](../scaffold-api/) — its README is the source of truth for prerequisites, scripts, and folder structure.

## What lives in this tree

| Subdir | Audience | Purpose |
|---|---|---|
| [`guides/`](guides/) | Developers using the scaffold | Role-based how-to guides (frontend, backend, QA, DevOps) and the manual smoke-test walkthrough |
| [`reference/`](reference/) | Architects, integration specialists | Deep reference catalogs: contract-first maturity model, EIP patterns, OpenAPI Overlay spec |
| [`product-specs/`](product-specs/) | Stakeholders, contributors | Product Requirements Documents — the WHY, the goals, the success metrics |
| [`exec-plans/`](exec-plans/) | Contributors maintaining the scaffold | Implementation plans broken into incremental, testable units |
| [`slide-decks/`](slide-decks/) | Presenters | The Data Brutalist slide deck, the demo delivery guide, and the slide build pipeline |
