### 1.2 Write the OpenAPI spec

Create `specs/order-api.yaml`. Build it progressively (you can show this progression in the video):

**Start with the info and servers block:**

```yaml
openapi: "3.1.0"
info:
  title: Order Management API
  version: 1.0.0
  description: |
    API for managing customer orders.
    Part of the API-First demo for the IT department.
  contact:
    name: API Team
    email: api-team@yourcompany.com

servers:
  - url: http://localhost:4010
    description: Mock server (Prism)
  - url: https://api.dev.yourcompany.com/v1
    description: Development
  - url: https://api.staging.yourcompany.com/v1
    description: Staging
  - url: https://api.yourcompany.com/v1
    description: Production
```

**Define security schemes:**

```yaml
security:
  - apiKey: []

components:
  securitySchemes:
    apiKey:
      type: apiKey
      in: header
      name: X-API-Key
    oauth2:
      type: oauth2
      flows:
        clientCredentials:
          tokenUrl: https://auth.yourcompany.com/oauth/token
          scopes:
            orders:read: Read orders
            orders:write: Create and update orders
```

**Define schemas with examples AND x-faker annotations:**

Write 3 to 4 schemas: Order, Customer, OrderItem, ErrorResponse. For each schema property, add:
- `type` and `format` (always)
- `example` (for static docs and screenshots)
- `x-faker` (for dynamic mocking)
- `description` (for generated docs)
- Constraints: `minimum`, `maximum`, `minLength`, `maxLength`, `pattern`, `enum`

Use the Order Management example from the interactive walkthrough we built earlier. Make sure the spec has at least these endpoints:
- `GET /orders` (list with pagination)
- `GET /orders/{orderId}` (single order)
- `POST /orders` (create)
- `PUT /orders/{orderId}` (update)
- `DELETE /orders/{orderId}` (cancel)
- `GET /orders/{orderId}/items` (list items)

For each endpoint, define:
- All parameters (path, query, header)
- Request body with schema reference
- At least two responses (success + error)
- `operationId` (needed for code generation)
- `tags` (for docs grouping)

**Add pagination to the list endpoint:**

```yaml
/orders:
  get:
    parameters:
      - name: page
        in: query
        schema:
          type: integer
          minimum: 1
          default: 1
      - name: limit
        in: query
        schema:
          type: integer
          minimum: 1
          maximum: 100
          default: 20
      - name: status
        in: query
        schema:
          type: string
          enum: [pending, confirmed, shipped, delivered, cancelled]
```

**Add OpenAPI links for stateful testing:**

```yaml
# In POST /orders response:
responses:
  "201":
    description: Order created
    content:
      application/json:
        schema:
          $ref: "#/components/schemas/Order"
    links:
      GetOrder:
        operationId: getOrder
        parameters:
          orderId: $response.body#/id
      UpdateOrder:
        operationId: updateOrder
        parameters:
          orderId: $response.body#/id
```

These links allow Schemathesis to do stateful testing: create an order, then use the returned ID to get/update it.

### 1.3 Validate the spec

```bash
# Check it parses correctly
npx @scalar/cli validate specs/order-api.yaml

# Quick visual check
npx @scalar/cli serve specs/order-api.yaml
# Opens a browser with rendered docs and built-in "Try it out"
```

Fix any errors before moving on. Every downstream tool depends on a valid spec.

## Phase 2: Set Up Linting with Spectral

**Time estimate: 1 hour**

### 2.1 Create the Spectral ruleset

Create `rules/.spectral.yaml`:

```yaml
extends:
  - spectral:oas

rules:
  # Enforce descriptions on all operations
  operation-description:
    severity: warn
    given: "$.paths[*][get,post,put,patch,delete]"
    then:
      field: description
      function: truthy

  # Require operationId on all operations
  operation-operationId:
    severity: error

  # Enforce camelCase for property names
  properties-camelCase:
    severity: warn
    given: "$.components.schemas[*].properties[*]~"
    then:
      function: casing
      functionOptions:
        type: camel

  # Require examples on schemas
  schema-examples:
    severity: warn
    given: "$.components.schemas[*].properties[*]"
    then:
      field: example
      function: truthy

  # Require error response on all operations
  must-have-error-response:
    severity: warn
    given: "$.paths[*][get,post,put,patch,delete].responses"
    then:
      - field: "4XX"
        function: truthy
```

### 2.2 Run Spectral and fix violations

```bash
spectral lint specs/order-api.yaml --ruleset rules/.spectral.yaml
```

For the demo, intentionally leave 2 to 3 violations in the spec so you can show Spectral catching them, then fix them live (or in a pre-recorded take).

### 2.3 Prepare a "bad spec" variant

Create `specs/order-api-bad.yaml` with intentional violations:
- A property named `order_status` (snake_case, violates camelCase rule)
- An endpoint missing `operationId`
- A schema property without a description or example

```yaml
    bindings:

        queue:

components:
  messages:
    OrderCreated:
      payload:
        type: object
        properties:
          orderId:
            type: string
            format: uuid
          customerId:
            type: string
          createdAt:
            type: string
            format: date-time
          items:
            type: array
            items:
              type: object
              properties:
                productId:
                  type: string
                quantity:
                  type: integer

    OrderStatusChanged:
      payload:
        type: object
        properties:
          orderId:
            type: string
            format: uuid
          previousStatus:
            type: string
            enum: [pending, confirmed, shipped, delivered]
          newStatus:
            type: string
            enum: [pending, confirmed, shipped, delivered, cancelled]
          changedAt:
```

## Phase 11: CI Pipeline Example

**Time estimate: 1 hour**

Create `ci/pipeline.yaml` (GitHub Actions format) to show the full automated pipeline:

```yaml
name: API Contract Pipeline
on:
  push:
    paths:
      - "specs/**"
  pull_request:
    paths:
      - "specs/**"

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @stoplight/spectral-cli
      - run: spectral lint specs/order-api.yaml --ruleset rules/.spectral.yaml

  breaking-changes:
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - run: |
          pip install oasdiff
          git show origin/main:specs/order-api.yaml > /tmp/old-spec.yaml
          oasdiff breaking /tmp/old-spec.yaml specs/order-api.yaml --fail-on ERR

  contract-test:
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @stoplight/prism-cli
      - run: prism mock -d specs/order-api.yaml --port 4010 &
      - run: sleep 3
      - uses: schemathesis/action@v2
        with:
          schema: specs/order-api.yaml
          base-url: http://localhost:4010
          checks: all

  generate-clients:
    runs-on: ubuntu-latest
    needs: contract-test
    steps:
      - uses: actions/checkout@v4
      - run: |
          npm install -g @openapitools/openapi-generator-cli
          openapi-generator-cli generate \
            -i specs/order-api.yaml \
            -g typescript-fetch \
            -o generated/client-typescript
      - uses: actions/upload-artifact@v4
        with:
          name: typescript-client
          path: generated/client-typescript/

  generate-docs:
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4
      - run: |
          npx @scalar/cli bundle specs/order-api.yaml -o docs/index.html
      - uses: actions/upload-artifact@v4
        with:
          name: api-docs
          path: docs/

  publish-apic:
    runs-on: ubuntu-latest
    needs: [contract-test, breaking-changes]
    if: github.ref == 'refs/heads/main'
    steps:
```
